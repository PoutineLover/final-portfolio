Recursion

Introduction

A factorial (!) is a mathematical operation that applies to integers. Given a number n, n factorial is the product of every number from 1 to n. For example, 3! is the same as 1 × 2 × 3, or 6. Could we write a program that could calculate the factorial of any number?

One way we could do this is by using a for loop to access each integer that appears before the user's input. 

 public static void main (String[] args){
    Scanner input = new Scanner(System.in);
    int factorial = input.nextInt();
    int product = 1;	
    for(int number = 1; number <= factorial; number++){
       product = product * number;		
    }
    System.out.println(factorial + " factorial is equivalent to: 
    " + product);
 }

This solution works, but there is another way to do it without using a loop: recursion

Recursion

Recursion is when an object is defined using a reference to itself. Some examples of this in real life are:

  • A bus stop is connected to a certain stop if it is either the requested stop in question, or if it is connected to another bus stop that is connected to the stop.
  • A company is owned by Google if it is Google, is owned by Google, or is owned by another company that itself is owned by Google.
  • An electric appliance is being charged if it is connected to an outlet, or if it is connected to another electric appliance that itself is connected to an outlet.

There are many examples of recursion in the real world than those I just mentioned. It may seem like a difficult concept at first, but it becomes easier to use the more you use it.

You may have noticed that despite the repetition in recursive definitions, there is eventually an end. For example, when checking if a bus stop is connected to a certain station, we will check for two things: First, we will check if the station that we are at is the station we are looking for. If it is, we're done; we determined that the bus stop is connected to the stop we're looking for. However, if we still haven't found our station, we will then check if another stop can be visited from our position, one that we haven't checked yet. If there is an unvisited stop, we will go to that stop, and perform the same set of actions that we just made on this current stop. But if there are no further stops to check, then we know that we cannot reach our target stop from our current position. One way or another, we will either realize that there is, or isn't, a path to our destination.

In programming, most recursive functions are formatted as methods. Thus, our factorial function would appear something like this: 

 public int factorial(int number){
   /*Implementation will follow*/
 }

Format Of A Recursive Method

In a recursive body, there is always a base case, and a recursive call. The base case is a condition, that if met, ends the recursive function. In our bus example, we actually have two base cases. Whether or not the station we we are checking is the destination, and if it isn't, whether there is another stop that we can check. Thus, the recursive function would end if we found the destination, or if we didn't find it and reached a dead end.

Recursive functions also have a recursive call. This is the part of the function that refers to itself. For example, if the bus stop we are currently checking is not our destination, but we have an adjacent station we could check, then the recursive call is the act of repeating the same inspection we just made, on our adjacent stop. If that stop also isn't our destination, but we still have another possible station we could check, then we would repeat the same process, and so on, until one of the base cases is met.

Using what we now know, we can attempt to create a method that will return the factorial of any number. Let's start with trying to find the base case of our recursive method. If we think about it, every factorial includes 1 factorial as a factor (see this for yourself). 1! is also the smallest possible factorial, and thus, we know that its value is always 1, without requiring knowledge of any other factorial. We can use this to set up our base case. 

 public int factorial(int number){
   if(number == 1){
      return 1;
   }
 }

What about our recursive call? We need to create a recursive call so that eventually, it will call factorial(1) so that our program can end. To do this, the integer parameter must decrease each time we call another iteration of factorial. At the same time, we want to somehow include the integer of our current call in our final product. We can accomplish both with the following lines of code: 

 public int factorial(int number){
   if(number == 1){
      return 1;
   }
   return number * factorial(number - 1);
 }

Now, our method has a base case, and it also has a recursive call. If we trace our method with an input of 4, we can see that the method returns the correct answer, 24:

Recursion Versus Loops

Recursion has some advantages over loops. First, recursion often provides greater clarity. After all, if you understand the base case and the recursive call, then you now know how the entire method works.

However, recursion also has it flaws. Space is created in the program’s storage, or “stack”, whenever a method is called, and is removed upon its completion. In the case of recursion, whose implementations include further calls to itself, the recursive method only ends when its “final” repetition finishes. Thus, the space required for a recursive function is proportional to the number of times it calls itself, which can lead to storage problems.

Conclusion

Recursion can be used in almost any situation that would regularly require a loop. However, when you choose to use it is up to you. You may decide that you value storage over clarity, and continue to use loops in programming. But if you like an elegant solution to your programming problems, then recursion may be the method of choice for you.
