Arrays & ArrayLists

Introduction

You are given the task of writing a program that will accept 10 integers and output their sum. How would you do this? You could create a field for each individual integer, and initialize them with the user's input, but that would be tedious and repetitive work. If this was in real life, you could easily record the integers onto a list and compute their sum afterwards; is there a way we could do that with programming?

Arrays

An array is the programming equivalent of a list; it is a single field that can store multiple elements, and you can also easily access each individual element inside. The declaration of an array looks like this: 

 Type [] identifier;

When we are initializing our array, we have to use the new keyword, because arrays are a type of object. Upon initialization, we also have to specify the length of our array, or how many elements it can store. For our example, let's initialize an array of integers called intList with a length of 10. 

 int [] intList = new int[10];

In an array, each individual element is assigned an index, which allows it to be accessed individually from other elements. In an array, the index of the first element is 0, the second element has an index of 1, the third has an index of 2, and so on. Essentially, for any element with index n, its position in the array is actually n + 1. It may seem a little confusing at first, but don't worry. It gets easier to remember this concept over time. The syntax for an element of an array (including initialization) looks like this: 

 intList[index] = value;

In order to assign a value to each element in the array, we could use a Scanner class to accept input and do this: 

 public static void main (String[] args){
    Scanner input = new Scanner (System.in);
    int [] intList = new int[10];   
    intList[0] = input.nextInt();
    intList[1] = input.nextInt();
    intList[2] = input.nextInt();
    intList[3] = input.nextInt();
    intList[4] = input.nextInt();
    intList[5] = input.nextInt();
    intList[6] = input.nextInt();
    intList[7] = input.nextInt();
    intList[8] = input.nextInt();
    intList[9] = input.nextInt();
 }

But aesthetically, it's hardly different from if we just created 10 separate fields in the first place. Also, if our array had a length of 100 elements, or 1000, manually assigning each value would be extremely annoying, not to mention boring. Instead, we can use a for loop, which can iterate through each position of our array: 

 public static void main (String[] args){
    Scanner input = new Scanner (System.in);
    int [] intList = new int[10];   
    for(int arrayPosition = 0; arrayPosition < intList.length; 
    arrayPosition++){
       intList[arrayPosition] = input.nextInt();
    }
 }

Now this code does exactly what it did before, but now it's a lot cleaner. Also, arrayPosition ends when its value is one less than the length of intList. Think about how the index of an element corresponds to its position in the array to see why this is true. As well, instead of using a literal as our end condition, we can use the field identifier.length, which returns the length of any given array, for added convenience.

We can then use another for loop to compute the sum of all the elements in the array, and output the sum afterwards. 

 public static void main (String[] args){
    Scanner input = new Scanner (System.in);
    int [] intList = new int[10];   
    for(int arrayPosition = 0; arrayPosition < intList.length;
    arrayPosition++){
       intList[arrayPosition] = input.nextInt();
    }
    int sum = 0;
    for(int arrayPosition = 0; arrayPosition < intList.length; 
    arrayPosition++){
       sum = sum + intList[arrayPosition];
    }
    System.out.println("The sum of all elements in the array 
    intList[] is: " + sum);
 }

There! Now we've used an array to efficiently store and manipulate a group of values!

ArrayLists

However, arrays have one fallacy: You can't adjust their length after they've been initialized. Thus, if we wanted to make a list to record the sum of an unknown amount of integers, an array wouldn't work. For example, imagine this problem: 

 Create a program that will continuously accept 
 integer inputs until the integer 0 is detected.
 When this integer is detected, the program will 
 stop accepting integers and output the sum.

To solve this problem, we would need an array that could change its size accordingly. This is a moment where we would use an ArrayList. ArrayLists are similar to arrays, except that ArrayLists don't have a given size. Thus, their size can change throughout the course of a program. The declaration and initialization of an ArrayList is as follows: this: 

 ArrayList<Type> identifier = new ArrayList<Type> ();

Since for our purposes we want to create an ArrayList of integers, our ArrayList initialization would resemble this: 

 ArrayList❬Integer❭ intArrayList = new ArrayList❬Integer❭ ();

The following syntax is used to add an element to the ArrayList: 

 identifier.add(value);

If the value entered is the only parameter, the value will be added to the end of the ArrayList. However, to insert a value into a specific index, another iteration of the add method can be called: 

 identifier.add(index, value);

If there is a value already present in the index where we want to insert our new value, the original value and all values after it are shifted one index to the right, and our new value is inserted into the now empty index.

If we want to access an element in our ArrayList, we have to use a special method called the get method. This method will allow us to access the value at a specific index of the ArrayList. 

 identifier.get(index);

Now that we've learned how to use an ArrayList, we can solve our example problem. First, we need to use the Scanner class to accept input and record them in an ArrayList. We also want to stop accepting input when the value inputted is 0. We can accomplish this using a while loop: 

 public static void main (String[] args){
    Scanner input = new Scanner (System.in);
    ArrayList ❬Integer❭ intArrayList = new ArrayList❬Integer❭();
    int num = input.nextInt();
    while(num != 0){
       intArrayList.add(num);
       num = input.nextInt();
    }
 }

Using a for loop, we can then output the sum using a method similar to the one we used in our array example. There is one difference however, instead of using length to get the size of our ArrayList, we use a method called size(). 

 public static void main (String[] args){
    Scanner input = new Scanner (System.in);
    ArrayList  intArrayList = new ArrayList();
    int num = input.nextInt();
    while(num != 0){
       intArrayList.add(num);
       num = input.nextInt();
    }
    int sum = 0;
    for(int position = 0; position < intArrayList.size(); 
    position++){
       sum = sum + intArrayList.get(position);
    }
    System.out.println("The sum of all elements in the ArrayList 
    intArrayList is: " + sum);
 }

If an ArrayList can do everything an array can but more, why don't we always use ArrayLists? Well, it turns out that in order to create an ArrayList, a lot more storage is required. Thus, you should try to keep your usage of ArrayLists down to a minimum, meaning that you should only use ArrayLists when arrays won't work.

Conclusion

In this lesson, you learned about Arrays and ArrayLists. Besides their purposes, we also learned the best way to initialize them, as well as how to access their individual elements. So the next time you find yourself facing a problem, and you feel that some sort of list is required, don't be afraid to apply your newly learned knowledge in Arrays and ArrayLists!
