OOP: Advanced

Introduction

In your very first lesson, you learned the basics of OOP, such as how classes can be used to create instances and more. However, there's still lots more to learn about OOP, as it possesses even more unique characteristics. In this section, we will discuss the concepts of Inheritance and Polymorphism. As well, we will cover the process of UML Project Design, which is used in the planning of OOP projects.

Inheritance

Let's revisit an old example, our Car class: 

 public class Car{
    String brand;
    String colour;
    int litreCapacity;
    String licensePlateNumber;
    int distanceTraveled;
    public Car(String c, String b, int lCapacity, int lPNumber){
       brand = b;
       colour = c;
       litreCapacity = lCapacity;
       licensePlateNumber = lPNumber;	
       distanceTraveled = 0;	   
    }
    public void driveDistance(){
       distanceTraveled = distanceTraveled + 50;
       litreCapacity = litreCapacity - 50;
    }
    public void changeColour(String c){
       colour = c;
    }
    public void changeLicensePlate(String lPN){
       licensePlateNumber = lPN;
    }
    public static void main(String[] args){
       Car ourCar = new Car("blue", "Honda", 250, "A1B 2C3");
    }
 }

There have been a few modifications in the fields and methods of our class, but nothing that doesn't describe a car. Looking at this, let's create a new class, called ElectricCar. 

 public class ElectricCar{
    String brand;
    String colour;
    int electricCapacity;
    String licensePlateNumber;
    int distanceTraveled;
    public Car(String c, String b, int eCapacity, int lPNumber){
       brand = b;
       colour = c;
       electricCapacity = eCapacity;
       licensePlateNumber = lPNumber;	
       distanceTraveled = 0;	   
    }
    public void driveDistance(){
       distanceTraveled = distanceTraveled + 50;
       electricCapacity = electricCapacity - 50;
    }
    public void changeColour(String c){
       colour = c;
    }
    public void changeLicensePlate(String lPN){
       licensePlateNumber = lPN;
    }
    public static void main(String[] args){
       ElectricCar ourCar = new ElectricCar("red", "Tesla", 
       250, "D4E 5F6");
    }
 }

Our ElectricCar class looks pretty similar to our Car class. In fact, it shares many of the same fields and methods. Instead of essentially rewriting the code from our Car class, is there another way to create our ElectricCar class? It turns out, there is, using a concept called Inheritance. For example, our ElectricCar class could also look like this: 

 public class ElectricCar extends Car{
    int electricCapacity;
    public ElectricCar(String c, String b, int eCapacity, 
    int lPNumber){
      super(b, c, 0, lPNumber);
      electricCapacity = eCapacity;   
    }
    public static void main(String[] args){
       ElectricCar ourCar = new ElectricCar("red", "Tesla", 
       250, "D4E 5F6");
    }
 }

Now our class has much less code, but still possesses the same fields and methods from before. This is because the ElectricCar class now extends the Car class. In real life, we can imagine that an electric car is an extension of a regular car; it possesses many of the same properties of a car, but has some added uniqueness (it runs on electricity, not gas). In programming, by extending the Car, our ElectricCar class gains access to its fields and methods. At the same time, the ElectricCar class has some unique attributes of its own: It has a new field, called electricCapacity.

Polymorphism

Polymorphism is the concept that a single object can represent multiple different things. For example, if we look at our Car class, we notice that the driveDistance method still only alters the value of litreCapacity. This is a problem for our ElectricCar class, which uses the field electricCapacity. We need to alter the driveDistance method in the ElectricCar class so that it acts appropriately with the electricCar fields. 

 public class ElectricCar extends Car{
    int electricCapacity;
    public ElectricCar(String c, String b, int eCapacity, 
    int lPNumber){
      super(b, c, 0, lPNumber);
      electricCapacity = eCapacity;   
    }
    public void driveDistance(){
      distanceTraveled = distanceTraveled + 50;
      electricCapacity = electricCapacity - 50;
    }
    public static void main(String[] args){
       ElectricCar ourCar = new ElectricCar("red", "Tesla", 
       250, "D4E 5F6");
    }
 }

By rewriting the driveDistance method in our ElectricCar class, we modified the method so that it performs a different action in the ElectricCar class. However, the driveDistance method still performs its original function in the Car class. This is an example of overriding methods, which is a form of Polymorphism. Now, the driveDistance method can perform multiple actions in different classes.

You may have noticed that our driveDistance() method only alters our distanceTraveled and electricCapacity fields by a flat amount of 50. But what if we wanted to drive a distance that wasn't 50 units long? We could rewrite our current distanceTraveled method to accept a parameter for the distance traveled, but we could also create another driveDistance method inside our class. 

 public class ElectricCar extends Car{
    int electricCapacity;
    public ElectricCar(String c, String b, int eCapacity, 
    int lPNumber){
      super(b, c, 0, lPNumber);
      electricCapacity = eCapacity;   
    }
    public void driveDistance(){
      distanceTraveled = distanceTraveled + 50;
      electricCapacity = electricCapacity - 50;
    }
    public void driveDistance(int dTraveled){
      distanceTraveled = distanceTraveled + dTraveled;
      electricCapacity = electricCapacity - dTraveled;
    }
    public static void main(String[] args){
       ElectricCar ourCar = new ElectricCar("red", "Tesla", 
       250, "D4E 5F6");
    }
 }

This brings up an issue: How will the program know which driveDistance method its supposed to run, if they both share the same name? Simple, whether or not the driveDistance method is accompanied by an integer parameter. This type of Polymorphism is called overloading methods, where multiple versions of a method exist within the same class.
